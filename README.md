# About this repositoryA demonstration of how to use LLVM's pass infrastructure to obfuscate code. This project provides two custom LLVM passes that perform simple arithmetic instruction substitution as a form of code obfuscation.# Passes Included* ChangePass (HelloWorld_change.cpp): Transforms addition instructions into subtraction instructions.* ObfuscatePass (HelloWorld_obfuscate.cpp): Transforms a + b into a - (-b).Both passes are registered under the pipeline name simple-obf.# Prerequisites* CMake >= 3.13* LLVM development libraries (with headers and cmake config files)* Clang (for compiling test source to IR and linking final binaries)* opt (LLVM optimizer tool, for loading and running passes)Ensure that clang, opt, and the cmake package are available in your environment.# Build```mkdir build && cd buildcmake ..cmake --build .```This will:1. Compile both pass plugins as shared modules.2. Compile test.c to LLVM IR (test.ll).3. Apply each pass to produce obfuscated IR files (test_change.ll, test_obfuscate.ll).4. Link the obfuscated IR into executables (app_change, app_obfuscate).All targets are built automatically with `cmake --build .`.# Manual UsageYou can also invoke the passes manually via opt:```# Compile source to IRclang -Xclang -disable-O0-optnone -O0 -S -emit-llvm \      -Xclang -disable-llvm-passes test.c -o test.ll# Apply ChangePassopt -load-pass-plugin=./libChangePass.so -passes=simple-obf \    test.ll -o test_change.ll# Apply ObfuscatePassopt -load-pass-plugin=./libObfuscatePass.so -passes=simple-obf \    test.ll -o test_obfuscate.ll# Compile obfuscated IR to executableclang -O0 -g test_change.ll -o app_changeclang -O0 -g test_obfuscate.ll -o app_obfuscate```> Note: On macOS, replace .so with .dylib; on Windows, use .dll.# Project Structure```LLVMPass/├── CMakeLists.txt├── HelloWorld_change.cpp       # ChangePass implementation├── HelloWorld_obfuscate.cpp    # ObfuscatePass implementation└── test.c                      # Sample source file```
